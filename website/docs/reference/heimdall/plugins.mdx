A `SchedulingProfile` composes the gateway's routing decision from scorer and picker plugins. Scorers assign a score to each candidate inference pod; the picker selects a destination from the scored candidates. Each plugin is declared once under `spec.plugins` and referenced (with an optional `weight`) from a profile's `pluginRefs`.

The following plugins are available.

{/* AUTO-GENERATED:aigateway-plugins START */}
**Scorers**

- `constant-scorer`
- `inflight-requests-scorer`
- `kv-cache-utilization-scorer`
- `kv-utilization-scorer`
- `prefix-cache-scorer`
- `running-requests-scorer`
- `token-length-scorer`
- `waiting-requests-scorer`

**Pickers**

- `max-score-picker`
- `random-picker`
- `weighted-random-picker`
{/* AUTO-GENERATED:aigateway-plugins END */}

## Plugin configuration

Most plugins take no configuration. The plugins below accept a `config` block on their `spec.plugins` entry.

### prefix-cache-scorer

{/* AUTO-GENERATED:aigateway-plugin-config-prefix-cache START */}
| Key | Type | Default | Options |
| --- | --- | --- | --- |
| `normalization` | string | `longestPrefix` | `longestPrefix`, `input` |
| `transform` | string | `linear` | `linear`, `logistic` |
| `k` | number | `14.0` |  |
| `x0` | number | `0.7` |  |
{/* AUTO-GENERATED:aigateway-plugin-config-prefix-cache END */}

The KV block size is sourced from each pod's `InferenceWorker` (`modelCard.kvCacheBlockSize`, read from the engine) and is not set on the scorer.
