Odin serves models on Kubernetes through two custom resources in the `odin.moreh.io/v1alpha1` API group: `InferenceService` (a running inference deployment) and `InferenceServiceTemplate` (a reusable spec fragment that an `InferenceService` can reference).

## InferenceService

An `InferenceService` describes a model deployment â€” the inference engine, the model, parallelism, and the pod templates that run it. The pod templates (`template`, `workerTemplate`) are standard Kubernetes [`PodTemplateSpec`](https://kubernetes.io/docs/reference/generated/kubernetes-api/v1.32/#podtemplatespec-v1-core) objects.
